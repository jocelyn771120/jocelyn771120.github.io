Q1：安装完之后打开boot2docker闪退

照理说下载完安装包之后直接运行boot2docker Start就可以进行配置，配置成功就可以正常ssh连接。但是我安装成功之后打开boot2docker Start直接给我闪退出来。

我的解决方案：把virtualbox卸载去官网下载最新版本的。
官网：https://www.virtualbox.org/wiki/Downloads

Q2：Error:exit status 255

上个问题解决了之后又有了新的问题，Boot2Docker Start点开之后加载了很久很久都没有加载成功，感觉半个世纪都过去了还没开始，就用了docker up发现提示了这个错误。于是就参考了http://cnodejs.org/topic/55a24267419f1e8a23a64367把当前的boot2docker vm删了又重装，成功了。（这篇文章就写的很详细基本解决了我所有问题但是真的找了好久啊）

WHCTF Easyhook

发表于 Sep 24 2017

做到一半没做出来的题目，看了下writeup，发现掉进坑里了，重做了一遍。

输入长度为19，开始随便输入19位字符串进行调试,本来以为是在sub_401240处但是调试的时候发现到此函数的时候输入的字符串已经经过加密了，就往上找断点。

发现经过WriteFile函数之后字符串已经经过加密了，跟进去进入00401000，再跟进去就是加密算法了。

当时调试了下以为只要经过这个加密然后把加密后的字符串和sub_401240处”this_is_not_the_flag”进行比较就能得到flag，但是当时也很奇怪为啥sub_401240处的字符串是20位的，于是就去掉最后一个g去写脚本发现跑出来根本不是，就一直卡在这里，今天看了wp之后发现其实这个加密算法后面还有一个函数，当时根本没注意。

这个循环是用来检测加密之后的字符串和0040A030处的字符串是否一样，所以说其实真正要进行对比的是0040A030处的字符串而不是”this_is_not_the_fla”。所以脚本如下： 

s="ajygkFm.\x7f_~-SV{8mLn"    
flag=[''for i in range(19)]    
for i in range(18,-1,-1):
    if i%2 !=0:
        flag[i] = chr((ord(s[i]) ^ i) + i)
    else:
        if(i>1):
            flag[i] = chr(ord(s[i-2]) ^ (i-2))

flag[18] = chr(ord(s[18]) ^ 0x13)
print ''.join(flag)

但是还要注意的是，第一个算法是从第二位开始的，第一位就是下面0040A030数据段中的第一个字母也就是a，所以合起来得到flag。

极路由的锐捷认证配置

发表于 Sep 24 2017

新学妹住进来了之后因为是数学的妹子，没有开锐捷，宿舍买了极路由学着配置了一下发一波网络设置。

进入hiwifi.com之后登陆到后台在互联网中选锐捷认证。

因为我们学校的ip分配是动态分配，所以在设置的时候IP网关之类的都默认0.0.0.0，如果说是静态分配的话需要填写正确参数，组播方式选择锐捷，DHCP脚本选择标准，DHCP方式选择认证后。

因为是动态IP分配，所以还要克隆下mac地址。先把路由器上的网线拔了插到电脑上，把无线的选项关了，连接成功之后再把网线拔了重新插回路由器上，进入后台。高级设置中MAC地址克隆点击进去，在第二个选项中选择就会自动复制物理地址，然后拷贝到手动输入中，保存。

在互联网中选择自动上网方式，点击进去，保存。此时就应该能连通互联网了。
