0x06 06_angr_symbolic_dynamic_memory

因为是动态分配内存，所以并不知道把数据放在哪一个地址。
但是，buffer0和buffer1两个指针都是在bss段上。

利用到的方法就是，buffer里放一个指针，然后这个指针指向未使用的内存块的地址（这里程序本身没有对buffer0和buffer1这两个指针进行初始化，就是说野指针），用这个内存块来存放数据，这样我们就不用去找在程序运行的时候数据该放在哪一个地址了，我们自己给数据找了一个位置放。

假设我们要跳过输入直接从scanf之后开始执行的话，我们必须保证操作的时候ebp和esp与正常程序运行的时候相同，可以看到add esp,0x10之后，esp的位置是和进入handle_user函数后sub esp 0x18（记住push ebp之后的0x18是给局部变量用的空间)之后的位置相同。

那么我们在写脚本的时候，在设置完程序的入口点为0x08048697之后，把ebp的值设置为此时esp的值，我们就还原了正常程序运行时栈上的状态，接下来要操作的数据就都可以在栈上完成。

所以接下来要考虑的事情，就是把符号变量放在栈上的哪一块。从上图我们可以看出我们输入的两个格式化字符串分别放在了ebp-0xc和ebp-0x10的位置上，所以我们在符号执行的时候就把我们的符号变量放在相同位置上就OK了。至于放数据的话，就是通过改变esp,因为有8个字节是需要padding掉的。

Angr学习之符号表达式和约束求解

发表于 Jul 10 2018

关于约束求解

参考文档：https://docs.angr.io/docs/solver.html
除了可以当作模拟器，Angr还有一个东西叫做符号变量，通过它执行算术运算产生AST（百度了一下AST，是一种抽象语法树的数据结构，稍微的看了一下感觉又是在重新预习编译原理），这个抽象运算树可以成为求解器的约束。

当给你一个做了运算操作的输出结果，那么输入是什么？这个问题就可以用Angr来解决。

关于Claripy

参考文档：https://docs.angr.io/docs/claripy.html

Claripy是angr的一个关于求解程序引擎的库，claripy ASTs提供一种联系具体值和符号表达的方式。

Claripy这个库总共提供了三种类型的AST：BV,FP,Bool，支持多种类型的数据，比如说bitvector，floating-point number,布尔操作。

0x2 02_angr_find_condition

explore()的参数相当于对到达路径起了约束作用。
find参数：程序会在找到对应find条件的states放在一个found list里面
avoid参数：与find类似，找到相应的states之后放在avoided lisit里面

import angr,sys

def main():

    pro=angr.Project("./02_angr_find_condition")

    entry_addr=pro.factory.entry_state()

    simulation=pro.factory.simgr(entry_addr)

    print "!"

    def success(state):
        stdout_res=state.posix.dumps(sys.stdout.fileno())
        return 'Good Job.' in stdout_res

    def fail(state):
        stdout_res=state.posix.dumps(sys.stdout.fileno())
        return 'Try again.' in stdout_res

    simulation.explore(find=success,avoid=fail)

    if simulation.found:
        solution=simulation.found[0]
        print "1"
        print solution.posix.dumps(sys.stdin.fileno())

    else:
        print "Error"

if __name__ == "__main__":

    main()

OpenCV的样本训练

什么是样本训练？

首先要知道数据挖掘有一个方法叫做分类。
分类是什么？就是把数据库中的数据映射到响应类别中。所以数据分类有两个步骤：建模和模型应用。我们通过分析来构造我们所需要的模型，比如说我们当遇到一个陌生人的时候我们要分辨是男是女，那么我们的大脑就会获取这个陌生人的特征信息，通过大脑中的模型去把这个人归类到男或者归类到女中去，所以是男（女）性所以要的特征信息所构成的数据就形成了训练集，训练集中的样本就是具备是男性的特征信息所组成的数据（我这么理解的）。
那么我所理解的样本训练就是：根据已经有的训练样本对一大堆未知数据进行分类，然后选出符合训练样本的样本，同时也可以预测未来数据的样本分类。
OpenCV样本训练：
OpenCV有自带创建样本的exe文件：opencv_createsamples.exe
同时也有自带的训练样本的exe文件：opencv_traincascade.exe和opencv_harrtraining.exe。

OpenCV人脸识别

OpenCV提供了Python接口，这里在做人脸识别的时候用到的是一个opencv自带的haar特征分类器，也就是说这个分类器已经把人脸的特征信息收集成训练样本，我们在识别人脸的时候就可以直接用haar特征分类器（实际上是一个xml文件），一般存放在opencv安装目录下\data\haarcascades目录下。
